Blog Setup

2018-09-14 Written by krejac
#hugo  #git  #github 

I love setting up something new as I really appreciate the opportunity to ‘nerd out’ and try out new stuff and learn. I have previousely dabbled with different types of static site generators, Jekyll, Octopress, Bashblog and Chisel to name a few. But this time I wanted to try out Hugo.

Hugo

Hugo was super easy to install with apt on Linux Mint and brew on macOS taking care of any potential dependencies (apt install hugo and brew install hugo respectively). That is far from the case with other static site generators. Theming worked well too. I cloned directly into theme-dir (note-to-self: you don’t need to use the name of the git repo as folder name). I did have some trouble being able to generate content (site worked fine otherwise) and I think that may be because I mixed up front matter formats.1

GitHub hosting

So GitHub hosting is not new to me. I actually moved an old Jekyll site out of .github.io (User Page) but then serving from docs/ no longer worked.2 So I had to find a workaround. Enter Hugo docs. The gist of it is to create a submodule within a repository, that the generated contents is put in, and that submodule is the repo from your GitHub User Page. Sounds a bit cumbersome, but it works pretty darn well.

But even though Hugo’s docs are really good I needed to read up on submodules elsewhere to get it to work.

Git

So now I have a submodule within git repo, which means that I put contents of one dir into two repos at once. One for site generation and one for hosting - which is quite nifty to know how to do.

Also as an aside during this I got a bit of training on git -rf and mashing stories. Oh, and got to bed way to late… :-/
